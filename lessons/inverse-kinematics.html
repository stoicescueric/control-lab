Inverse Kinematics

"Put the claw there." Working out the joint angles that make a point in space happen.

Lots of FTC/FRC robots have a jointed arm: a shoulder, an elbow, a claw on the end. You don't think in joint angles — you think "reach the high junction" or "grab the sample at that spot." Turning a target position into the joint angles that get you there is inverse kinematics (IK). It's the same "go to a point" goal as path following, but for an arm instead of a drivetrain.

Forward first: angles → position

The easy direction is forward kinematics: if you know the angles, where's the hand? For a two-joint arm with link lengths L₁ and L₂, shoulder angle θ₁, and elbow angle θ₂ (measured relative to the first link), you just add up the two links:

x = L₁·cos(θ₁) + L₂·cos(θ₁ + θ₂)
y = L₁·sin(θ₁) + L₂·sin(θ₁ + θ₂)

The hard, useful direction: position → angles

Now run it backwards. Given a target (x, y), find θ₁ and θ₂. This is where a little triangle geometry — the law of cosines — does all the work. Picture the two links plus the straight line from the shoulder to the target: that's a triangle with sides L₁, L₂, and r (the distance to the target).

  1. Distance to the target: r² = x² + y².
  2. The elbow angle comes straight from the law of cosines:
cos(θ₂) = (r² − L₁² − L₂²) / (2·L₁·L₂)
θ₂ = ± arccos( … )  ← two answers: elbow-up or elbow-down
  1. Then the shoulder angle: aim the whole arm at the target, then back off by the angle the second link contributes.
θ₁ = atan2(y, x) − atan2( L₂·sin(θ₂),  L₁ + L₂·cos(θ₂) )
✌️
Two solutions, always. Your elbow can bend two ways to reach the same point — "elbow up" and "elbow down" (the ± on θ₂). Real robots pick whichever avoids hitting the robot, the floor, or the game element. Toggle it in the demo and watch the arm flip.
🧭
Why atan2 and not plain atan? atan2(y, x) knows which quadrant the target is in, so it gives the correct angle all the way around the circle (behind the robot, above, below). Plain atan(y/x) can't tell left from right and will silently point the arm the wrong way. Always use atan2 for directions.

Can it even reach?

A two-link arm can only touch points inside a ring: no further than L₁ + L₂ (arm straight out) and no closer than |L₁ − L₂| (arm folded back). Ask for a point outside that ring and cos(θ₂) comes out beyond ±1 — the math's way of saying "impossible." Good code checks for this; the demo shades the reachable ring and shows what happens when you drag the target past it.

static final double L1 = 0.45, L2 = 0.40;   // link lengths, metres

// Returns {shoulder, elbow} radians, or null if the point is out of reach.
double[] solveIK(double x, double y, boolean elbowDown) {
    double r2   = x*x + y*y;
    double cos2 = (r2 - L1*L1 - L2*L2) / (2 * L1 * L2);
    if (cos2 < -1 || cos2 > 1) return null;          // unreachable

    double theta2 = Math.acos(cos2);
    if (elbowDown) theta2 = -theta2;                    // the other solution

    double theta1 = Math.atan2(y, x)
                  - Math.atan2(L2 * Math.sin(theta2), L1 + L2 * Math.cos(theta2));
    return new double[] { theta1, theta2 };
}

// In your loop: solve, then send each joint to its angle.
double[] goal = solveIK(targetX, targetY, /*elbowDown=*/ false);
if (goal != null) {
    shoulder.setGoal(goal[0]);   // a ProfiledPIDController (Lesson 13)
    elbow.setGoal(goal[1]);      //   + ArmFeedforward kG·cos (Lesson 12)
}
🔗
This is where the whole track comes together. IK tells each joint what angle to hit. A motion profile (Lesson 13) gets it there smoothly, feedforward (Lesson 12) cancels gravity with kG·cos(θ), and PID (Lesson 8) cleans up the rest. Five lessons, one arm.

You made it — what now?

You've gone from "what is feedback?" to estimating a robot's pose and driving it along a planned path with a profiled, feed-forwarded arm on top. That's the real backbone of a competitive FTC/FRC robot. The honest next step is the fun one: open your robot project and start typing. Pick one mechanism, get ticks→metres right, add feedforward, then a profile, and watch it come alive. The rest of the understanding arrives the moment it moves.
