The Moving Average

The filter you already know — just average the last few readings.

You've used this one without knowing its name. A moving average (also called a rolling or sliding average) keeps the last N readings and reports their average. As each new reading arrives, the oldest one falls off the back — the window slides along.

📈 You've seen this. "7-day average of new cases", a "50-day stock average", your phone's average step count — all moving averages. They turn a spiky mess into a calm, readable trend.

How it works

output = ( x[now] + x[now-1] + … + x[now-N+1] ) / N   ← the average of the last N readings

Moving average vs. low-pass

They're cousins that solve the same problem two ways:

Try it — grow the window

A bigger window N averages more readings, so it's smoother — but it also reacts more slowly, because old readings keep dragging it back. Same trade-off as always: smooth vs. quick.

⚠️
Watch out: with a big window, hit "Sudden jump" — the green line crawls up like a ramp instead of snapping. A moving average is sluggish after sharp changes, and a single wild spike still nudges the average for N whole steps. For fast-moving or spiky data, the Kalman filter (two lessons ahead) is far smarter.
