# References & Resources

<Abstract>
This page collects the external sources the curriculum draws on, grouped by kind, plus the best places
to learn more. Each finished lesson should also carry its own local "Sources & references" admonition;
this page is the master list.
</Abstract>

Control Lab is built on open FTC/FRC documentation, control-theory literature, public team code, and
carefully cited community writing. If a technique in a lesson helped you, the original source below is
usually the best place to go deeper.

## Start here: community guides

- **[Game Manual 0](https://gm0.org/en/latest/)** - FTC hardware and software reference. Used for motor,
  drivetrain, odometry, mecanum, and general FTC software context.
- **[CTRL ALT FTC](https://www.ctrlaltftc.com/)** - focused FTC control-theory explanations for PID,
  feedforward, filtering, and tuning.
- **[FTC Tech Toolbox](https://ftc-tech-toolbox.vercel.app/)** - FTC autonomous movement walkthroughs and
  path-following framing.
- **[Controls Engineering in FRC](https://file.tavsys.net/control/controls-engineering-in-frc.pdf)**
  by Tyler Veness - rigorous open textbook for state-space control, feedback, modeling, and estimation in
  competitive robotics.

## Libraries & official documentation

- **[FTC SDK / FtcRobotController](https://github.com/FIRST-Tech-Challenge/FtcRobotController)** - the
  official Android-based robot controller and API.
- **[WPILib](https://docs.wpilib.org/)** - FRC standard library and reference documentation for PID,
  feedforward, motion profiles, state-space control, observers, and command-based programming.
- **[WPILib Command-Based Programming](https://docs.wpilib.org/en/stable/docs/software/commandbased/index.html)** -
  the original command architecture reference that many FTC frameworks adapt.
- **[FTCLib](https://docs.ftclib.org/)** - FTC library; its `PIDFController` and command-based framework
  appear in the [PID](./control-theory/pid.mdx) and [Command-Based](./software-architecture/command-based.mdx)
  lessons.
- **[SolversLib](https://docs.seattlesolvers.com/)** - maintained FTCLib-style command-based framework.
- **[Mercurial](https://docs.dairy.foundation/Mercurial/overview)** - command-based framework for advanced
  FTC teams using lambda-style command composition.
- **[Ivy by Pedro Pathing](https://pedropathing.com/docs/ivy)** - lightweight command-based control flow
  for composing Pedro paths with mechanism actions.
- **[Road Runner](https://rr.brott.dev/)** - FTC motion-planning library; its `Pose2d.exp` and spline
  material support the pose-exponential and spline lessons.
- **[Pedro Pathing](https://pedropathing.com/)** - reactive FTC path follower, visualizer, and Ivy command
  framework.
- **[goBILDA](https://www.gobilda.com/)** and **[REV Robotics](https://www.revrobotics.com/)** - vendor
  hardware references used when anchoring motor, encoder, and mechanism examples.

## Papers & articles

- Kevin Lynch and Frank Park, **[Modern Robotics](https://modernrobotics.northwestern.edu/)** - open
  robotics textbook and video course used for forward kinematics, inverse kinematics, Jacobians, and
  coordinate-frame reasoning.
- R. C. Coulter, **[Implementation of the Pure Pursuit Path Tracking Algorithm](https://www.ri.cmu.edu/pub_files/pub3/coulter_r_craig_1992_1/coulter_r_craig_1992_1.pdf)**,
  CMU Robotics Institute, 1992 - original pure-pursuit derivation.
- Team 1712 DAWGMA, **[Implementation of the Adaptive Pure Pursuit Controller](https://www.chiefdelphi.com/t/paper-implementation-of-the-adaptive-pure-pursuit-controller/166552)** -
  curvature velocity profile and inject-and-smooth path generation.
- Timo Denk, **[Cubic Spline Interpolation](https://blog.timodenk.com/cubic-spline-interpolation/)** -
  natural cubic spline continuity conditions and tridiagonal solve.
- Vikramaditya Nishant, **[Introducing Torque-Current Control for FTC](https://medium.com/@vikramaditya.nishant/introducing-torque-current-control-for-ftc-3db75d281258)** -
  current, voltage, back-EMF, and current-limiting discussion.
- E. Eade, *Lie Groups for 2D and 3D Transformations* - background for the $SE(2)$ exponential/log maps.
- Kou, E. & Haggenmiller, A., **[Extended Kalman Filter State Estimation for Autonomous Competition Robots](https://arxiv.org/abs/2310.04459)** -
  FTC-oriented EKF paper used by the [Kalman](./signal-processing/kalman.mdx) and
  [EKF](./signal-processing/ekf.mdx) lessons.
- Y. A. Kapitanyuk, A. V. Proskurnikov & M. Cao,
  **[A Guiding Vector Field Algorithm for Path Following Control](https://arxiv.org/abs/1610.04391)** -
  implicit/level-set GVF formulation and Lyapunov convergence proof.
- Havish Sripada and Baron Henderson, **Second-Order Multidimensional Guiding Vector Field for Parametric
  Path-Following Under Holonomic Constraints**, Pedro Pathing, 2026 - second-order GVF framework behind
  Pedro Pathing. User-supplied PDF.

## Source code referenced

Open-source FTC team code the lessons translate. All code in Control Lab is an independent
reimplementation, not a copy.

- **[petrustoica/CenterStage](https://github.com/petrustoica/CenterStage)** - cubic Bezier path follower.
- **[SimplicityFTC/SimplicityFTC-Quickstart](https://github.com/SimplicityFTC/SimplicityFTC-Quickstart)** -
  PID-to-point, kStatic, and point-advance sequencing.
- **[Cyliis/CyLiis_Into_The_Deep_OfficialCode](https://github.com/Cyliis/CyLiis_Into_The_Deep_OfficialCode)** -
  predictive deceleration through a glide-distance predicted pose.
- **[FTCclueless/Centerstage](https://github.com/FTCclueless/Centerstage)** - kStatic, spline following,
  curvature speed caps, centripetal feedforward, and line-circle lookahead.
- **[KHEngineering/SmoothPathPlanner](https://github.com/KHEngineering/SmoothPathPlanner)** - gradient-descent
  path smoother from Team 2168.
- **[xiaoxiae/PurePursuitAlgorithm](https://github.com/xiaoxiae/PurePursuitAlgorithm)** - adaptive pure-pursuit
  visualizer.
- **[FTC-9974-THOR/GuidedVectorField](https://github.com/FTC-9974-THOR/GuidedVectorField)** - reference FTC
  guided-vector-field library.
- **[KookyBotz/CenterStage `GVFCommand`](https://github.com/KookyBotz/CenterStage/blob/master/TeamCode/src/main/java/org/firstinspires/ftc/teamcode/common/commandbase/drivecommand/GVFCommand.java)** -
  shipped GVF follower with projection, gains, curvature speed cap, and motion-profiled heading.
- WolfPack Machina Power Play autonomous example supplied by the site author - representative FSM excerpt
  credited in [State Machines](./software-architecture/state-machines.mdx). Contact: jkc.cassidy+FTC@gmail.com.
