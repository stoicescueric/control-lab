# References & Resources

<Abstract>
This page collects every external source the curriculum draws on, grouped by kind, plus the best places
to learn more. Each lesson also carries its own "Sources & references" admonition; this is the master
list. Everything here is freely available online.
</Abstract>

Control Lab is built on the shoulders of the FTC/FRC community's open documentation, the control-theory
literature, and a handful of teams who published their code. If a technique in a lesson helped you, the
original source below will take you deeper.

## Start here — community guides

The fastest on-ramps to FTC software, and the references this site leans on most.

- **[Game Manual 0 (gm0)](https://gm0.org/en/latest/)** — the community encyclopedia of FTC hardware and
  software. Used directly for the [motor model](./control-theory/how-a-motor-works.mdx)
  ([motor power guide](https://gm0.org/en/latest/docs/power-and-electronics/motor-guide/motor-power.html))
  and [mecanum kinematics](./path-following/mecanum-kinematics.mdx)
  ([mecanum drive](https://gm0.org/en/latest/docs/software/tutorials/mecanum-drive.html)).
- **[CTRL ALT FTC](https://www.ctrlaltftc.com/)** — a focused, math-forward guide to PID, feedforward, and
  control for FTC.
- **[FTC Tech Toolbox](https://ftc-tech-toolbox.vercel.app/)** — autonomous-movement walkthroughs; an
  influence on the [Path Following](./path-following/index.mdx) module's framing.
- **[Controls Engineering in FRC](https://file.tavsys.net/control/controls-engineering-in-frc.pdf)**
  (Tyler Veness) — a rigorous, free textbook on state-space control for competitive robotics.

## Libraries & official documentation

- **[FTC SDK / FtcRobotController](https://github.com/FIRST-Tech-Challenge/FtcRobotController)** — the
  official Android-based robot controller and API.
- **[FTCLib](https://docs.ftclib.org/)** — community library; its `PIDFController` and command-based
  framework appear in the [PID](./control-theory/pid.mdx) and
  [command-based](./software-architecture/command-based.mdx) lessons.
- **[Road Runner](https://rr.brott.dev/)** — FTC motion-planning library; its `Pose2d.exp`/`splineTo`
  back the [pose exponential](./localization-odometry/pose-exponential.mdx) and
  [spline](./path-following/quintic-splines.mdx) lessons.
- **[Pedro Pathing](https://pedropathing.com/)** — a reactive FTC path follower (Bézier + heading
  control).
- **[WPILib](https://docs.wpilib.org/)** — the FRC standard library; source of `Pose2d.exp(Twist2d)`,
  `SimpleMotorFeedforward`, and **SysId** (referenced in
  [system identification](./control-theory/system-identification.mdx)).
- **[goBILDA](https://www.gobilda.com/)** & **[REV Robotics](https://www.revrobotics.com/)** — motors and
  the hardware the lessons anchor to (e.g. the goBILDA 5000-0002-0117 motor curve).

## Papers & articles

- R. C. Coulter, **[Implementation of the Pure Pursuit Path Tracking Algorithm](https://www.ri.cmu.edu/pub_files/pub3/coulter_r_craig_1992_1/coulter_r_craig_1992_1.pdf)**,
  CMU Robotics Institute, 1992 — the original pure-pursuit derivation
  ([Pure Pursuit](./path-following/pure-pursuit.mdx)).
- Team 1712 (DAWGMA), **[Implementation of the Adaptive Pure Pursuit Controller](https://www.chiefdelphi.com/t/paper-implementation-of-the-adaptive-pure-pursuit-controller/166552)**
  — the whitepaper behind the curvature velocity profile and inject-and-smooth path generation.
- Timo Denk, **[Cubic Spline Interpolation](https://blog.timodenk.com/cubic-spline-interpolation/)** — the
  natural cubic spline tridiagonal derivation ([Splines](./path-following/quintic-splines.mdx)).
- Vikramaditya Nishant, **[Introducing Torque-Current Control for FTC](https://medium.com/@vikramaditya.nishant/introducing-torque-current-control-for-ftc-3db75d281258)**
  — the $V = k_e\omega + IR$ current-command and current-limiting argument
  ([How a Motor Works](./control-theory/how-a-motor-works.mdx)).
- E. Eade, *Lie Groups for 2D and 3D Transformations* — background for the $SE(2)$ exponential/log maps
  ([Pose Exponential](./localization-odometry/pose-exponential.mdx)).
- Y. A. Kapitanyuk, A. V. Proskurnikov & M. Cao, **[A Guiding Vector Field Algorithm for Path Following Control](https://arxiv.org/abs/1610.04391)**
  — the implicit/level-set field $\chi = E\nabla\varphi - k\varphi\nabla\varphi$ and its Lyapunov
  convergence proof ([Guided Vector Fields](./path-following/guided-vector-fields.mdx)).

## Source code referenced

Open-source FTC team code the lessons translate, each with the technique it anchors. All code in Control
Lab is an independent reimplementation, not a copy.

- **[petrustoica/CenterStage](https://github.com/petrustoica/CenterStage)** — cubic Bézier path follower.
- **[SimplicityFTC/SimplicityFTC-Quickstart](https://github.com/SimplicityFTC/SimplicityFTC-Quickstart)** —
  PID-to-point, kStatic, point-advance sequencing.
- **[Cyliis/CyLiis_Into_The_Deep_OfficialCode](https://github.com/Cyliis/CyLiis_Into_The_Deep_OfficialCode)** —
  predictive deceleration (glide-distance predicted pose).
- **[FTCclueless/Centerstage](https://github.com/FTCclueless/Centerstage)** — kStatic, spline following
  (curvature speed + centripetal feedforward), line–circle lookahead.
- **[KHEngineering/SmoothPathPlanner](https://github.com/KHEngineering/SmoothPathPlanner)** — the
  gradient-descent path smoother (Team 2168).
- **[xiaoxiae/PurePursuitAlgorithm](https://github.com/xiaoxiae/PurePursuitAlgorithm)** — an adaptive
  pure-pursuit visualizer.
- **[FTC-9974-THOR/GuidedVectorField](https://github.com/FTC-9974-THOR/GuidedVectorField)** — a reference
  FTC guided-vector-field library (Bézier-path guidance, disturbance rejection).
- **[KookyBotz/CenterStage `GVFCommand`](https://github.com/KookyBotz/CenterStage/blob/master/TeamCode/src/main/java/org/firstinspires/ftc/teamcode/common/commandbase/drivecommand/GVFCommand.java)**
  — a shipped GVF follower: $k_N/k_S/k_C$ gains, curvature speed cap, seeded projection, motion-profiled
  heading ([Guided Vector Fields](./path-following/guided-vector-fields.mdx)).
